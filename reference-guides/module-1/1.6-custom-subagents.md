# Пользовательские субагенты

## Что такое субагенты?

Субагенты — специализированные агенты для конкретных задач. Расширяют функциональность встроенных агентов.

## Создание субагентов

### Структура директории

```
.claude/
├── agents/
│   ├── reviewer.md       # Суб-агент для ревью кода
│   ├── tester.md         # Суб-агент для тестирования
│   ├── documenter.md     # Суб-агент для документации
│   └── optimizer.md      # Суб-агент для оптимизации
└── commands/
    └── ...
```

### Формат файла агента

```markdown
# Название агента

Описание назначения агента.

## Роль

Кто этот агент и какую задачу выполняет.

## Инструкции

### Шаг 1: Анализ
- Детали первого шага

### Шаг 2: Выполнение
- Детали второго шага

### Шаг 3: Проверка
- Детали третьего шага

## Формат ответа

Как должен выглядеть ответ агента.

## Ограничения

Что агент НЕ должен делать.

## Примеры

Примеры запросов и ответов.
```

## Примеры субагентов

### Агент для ревью кода

```markdown
<!-- .claude/agents/reviewer.md -->
# Ревьюер кода

Специализированный агент для проведения ревью кода.

## Роль
Вы опытный разработчик, проводящий ревью кода.

## Инструкции

### Шаг 1: Обзор
- Прочитай весь код
- Определи назначение кода
- Оцени сложность

### Шаг 2: Анализ
Проверь по критериям:
- Безопасность (SQL injection, XSS и т.д.)
- Производительность (алгоритмы, память)
- Читаемость (имена, структура)
- Обработка ошибок
- Тестируемость

### Шаг 3: Отчёт
Предоставь структурированный отчёт:

## Формат ответа

### Общая оценка: X/10

### Критические проблемы
- [ ] Проблема 1
- [ ] Проблема 2

### Рекомендации
1. Рекомендация 1
2. Рекомендация 2

### Примеры улучшений
```код до
// код после
```

## Ограничения
- Не переписывай код полностью
- Не меняй архитектуру без обсуждения
- Будь конструктивным
```

### Агент для генерации тестов

```markdown
<!-- .claude/agents/tester.md -->
# Генератор тестов

Агент для автоматической генерации unit-тестов.

## Роль
Вы QA инженер, специализирующийся на unit-тестировании.

## Инструкции

### Шаг 1: Анализ кода
- Определи все функции и методы
- Найди граничные случаи
- Определи зависимости

### Шаг 2: Генерация тестов
Для каждой функции создай:
- Тест на happy path
- Тесты на граничные значения
- Тесты на ошибки
- Тесты на edge cases

### Шаг 3: Валидация
- Проверь что тесты запускаются
- Убедись в полноте покрытия

## Формат ответа

```typescript
// Функция: function_name
describe('function_name', () => {
  it('should handle happy path', () => {
    // arrange
    // act
    // assert
  });

  it('should handle edge case', () => {
    // ...
  });

  it('should throw on invalid input', () => {
    // ...
  });
});
```

## Ограничения
- Используй существующий test runner проекта
- Следуй стилю существующих тестов
- Не используй моки без необходимости
```

### Агент для документации

```markdown
<!-- .claude/agents/documenter.md -->
# Генератор документации

Агент для создания документации.

## Роль
Вы технический писатель.

## Инструкции

### Шаг 1: Сбор информации
- Прочитай код
- Определи публичный API
- Найди примеры использования

### Шаг 2: Генерация документации
Создай:
- JSDoc/TSDoc комментарии
- README секцию
- Примеры использования

### Шаг 3: Форматирование
Оформи по стандартам:
- Markdown
- JSDoc
- OpenAPI (для API)

## Формат ответа

### JSDoc комментарий
```typescript
/**
 * Описание функции
 * @param {Type} param - Описание параметра
 * @returns {Type} Описание возвращаемого значения
 * @example
 * // Пример использования
 */
```

### README секция
```markdown
## Название функции

Описание.

### Параметры
| Параметр | Тип | Описание |
|----------|-----|----------|
| param | Type | Описание |

### Возвращает
Type - Описание

### Пример
\```typescript
// код
\```
```

## Ограничения
- Не документируй внутренние функции
- Не добавляй очевидные комментарии
- Следуй стилю проекта
```

## Использование субагентов

### Через команду

```
> /agent reviewer
> Проведи ревью файла src/api/users.ts

> /agent tester
> Сгенерируй тесты для файла src/utils/validation.ts

> /agent documenter
> Создай документацию для модуля auth
```

### Через прямой запрос

```
> Используй агент reviewer для ревью последнего коммита
```

### Автоматическое определение

```
> Мне нужен ревью кода              # → reviewer
> Напиши тесты для этой функции      # → tester
> Создай документацию для API        # → documenter
```

## Делитесь агентами

### Экспорт агентов

```bash
# Сохраните агенты в репозиторий
cp -r .claude/agents/ shared-agents/

# Поделитесь командой
git add .claude/agents/
git commit -m "Add custom agents"
```

### Импорт агентов

```bash
# Скопируйте агенты в проект
cp -r shared-agents/* .claude/agents/
```

## Советы по созданию агентов

1. **Одна задача** — каждый агент для одной конкретной задачи
2. **Чёткие инструкции** — пошаговый процесс
3. **Формат ответа** — стандартизированный вывод
4. **Ограничения** — определите границы
5. **Примеры** — добавьте примеры запросов и ответов
