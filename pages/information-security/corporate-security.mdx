## 3.3: Корпоративная безопасность

- **Время прохождения:** 20 минут

> **Интерактивный урок:** Запустите `/start-3-3` в OpenCode

## Обзор

Практические правила работы с файлами и промптами: как давать ИИ только необходимый контекст и проверять данные перед отправкой.

**Ключевой вывод:** ИИ не нужна вся информация о проекте. Давайте только то, что необходимо для задачи.

## Принцип минимального контекста

### Плохо
> «Проанализируй продажи нашего филиала на Арбате за февраль. Менеджер — Сидоров Алексей. Средний чек 850₽, у Starbucks — 920₽.»

Что попало: адрес, имя, внутренняя метрика, данные конкурента.

### Хорошо
> «Проанализируй данные по продажам кофейни (файл sales-feb.csv). Определи тренды и предложи 3 способа повысить средний чек.»

Тот же результат, но без чувствительных данных.

## Проверка файлов перед отправкой

| Тип файла | Что проверять |
|-----------|---------------|
| **Excel** | Скрытые листы, формулы со ссылками, комментарии, метаданные автора |
| **PDF** | Метаданные, скрытые слои, комментарии, цифровые подписи |
| **Word** | История изменений, комментарии, свойства документа |
| **Изображения** | EXIF-данные (геолокация, устройство) |
| **CSV** | Проверь все столбцы — в данных может быть неожиданное |

## Безопасные шаблоны промптов

**Анализ данных:**
> «Проанализируй данные из файла [файл]. Определи тренды и аномалии. Данные обезличены.»

**Создание документа:**
> «Создай шаблон отчёта по продажам. Структура: заголовок, период, ключевые метрики, выводы. Без реальных данных.»

**Код:**
> «Напиши функцию для расчёта конверсии. Вход: кол-во посещений (int), кол-во покупок (int). Возвращает float.»

## Чек-лист перед работой с ИИ

- [ ] Данные обезличены
- [ ] Файл проверен на скрытые данные
- [ ] В запросе только необходимая информация
- [ ] Нет паролей, ключей, токенов
- [ ] Нет юридически чувствительной информации
- [ ] Если сомневаюсь — спросил у руководителя или ИБ
