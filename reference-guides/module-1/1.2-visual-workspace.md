# Рабочее пространство AI-агента в VS Code

## Интерфейс

AI-агент живёт в боковой панели VS Code как расширение Claude Code. Когда ты кликаешь по его иконке слева — справа открывается панель чата.

```
┌───────────────┬────────────────────┬────────────────────────┐
│   Explorer    │    Редактор        │   AI-агент (панель)    │
│               │                    │                        │
│  📂 курс      │   # SCENARIO.md    │  > Твой запрос...      │
│    📁 docs    │                    │                        │
│    📁 src     │   Basecamp Coffee  │  ⬡ Ответ агента        │
│      📄 x.ts  │   — сеть из 45...  │                        │
│               │                    │                        │
└───────────────┴────────────────────┴────────────────────────┘
```

## Три зоны экрана

### Слева — Explorer
- Дерево файлов проекта.
- Открыть/свернуть: `Cmd/Ctrl+Shift+E`.
- Двойной клик по файлу открывает его в редакторе.

### В центре — Редактор
- Файлы, которые ты открыл.
- Несколько вкладок сразу.
- Markdown: превью через `Cmd/Ctrl+Shift+V` или правой кнопкой → «Open Preview».

### Справа — AI-агент
- Чат с агентом.
- Агент видит все файлы проекта и сам решает, какие открыть для ответа.
- История сессии сохраняется, пока открыт проект.

## Как общаться с агентом

### Простой вопрос
```
Что делает этот проект? Объясни структуру.
```

### Указать конкретный файл через @
```
@company-context/SCENARIO.md прочитай и дай саммари
```
Символ `@` открывает быстрый поиск файлов в проекте — агент получит точный файл в контекст.

### Запросить действие
```
Создай файл README.md с описанием из SCENARIO.md.
```

### Попросить diff
```
Покажи, что изменилось в последнем коммите.
```

## Полезные горячие клавиши

| Клавиша | Действие |
|---|---|
| `Cmd/Ctrl+Shift+E` | Открыть/свернуть Explorer |
| `Cmd/Ctrl+Shift+V` | Preview Markdown |
| `Cmd/Ctrl+P` | Quick Open — найти файл по имени |
| `Cmd/Ctrl+Shift+P` | Command Palette — любая команда VS Code |
| `` Cmd/Ctrl+` `` | Открыть/свернуть терминал |

Переключаться между Explorer, редактором и панелью агента удобнее мышкой, но `F6` циклически ходит между панелями если хочешь с клавиатуры.

## Настройка

Настройки Claude Code лежат в `settings.json` VS Code. Открыть: `Cmd/Ctrl+Shift+P → Preferences: Open User Settings (JSON)`.

Базовый блок (уже должен быть добавлен по инструкции из README):

```json
{
  "claudeCode.preferredLocation": "panel",
  "claudeCode.environmentVariables": [
    { "name": "ANTHROPIC_BASE_URL",   "value": "https://llm-gateway.agima.tech" },
    { "name": "ANTHROPIC_AUTH_TOKEN", "value": "YOUR_PERSONAL_TOKEN" }
  ]
}
```

Всё — этого хватает. Модель Gateway подбирает сам, локальных конфиг-файлов для агента не нужно.

## Советы по эффективности

1. **Используй `@`** — это быстрее, чем копировать пути вручную.
2. **Пиши контекст**, а не только вопрос. «Опиши проект» → слабый ответ. «Опиши проект: каталог папок, основные файлы, кто запускается первым» → сильный.
3. **Читай ответы агента** — он часто объясняет, почему сделал именно так. Если не согласен — возрази, он поправит.
4. **Не бойся сломать** — всё в git, откатить легко. Смелее экспериментируй с запросами.
5. **Если агент молчит или выдаёт ошибку** — проверь, что Gateway доступен: открой в браузере `https://llm-gateway.agima.tech/health` или напиши в `#ai-platform`.

## Что дальше

Ты умеешь видеть файлы и общаться с агентом. В следующем уроке — работа с файлами через `@`, быстрые паттерны чтения/записи и первые «реальные» задачи.
