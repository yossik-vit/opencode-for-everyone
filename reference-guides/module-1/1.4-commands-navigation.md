# Команды и навигация

## Slash-команды в панели агента

Набери `/` в поле ввода — откроется меню команд. Основные:

| Команда | Что делает |
|---|---|
| `/clear` | Очистить историю чата, начать с нуля |
| `/compact` | Сжать длинный разговор, сохранив суть |
| `/context` | Какие файлы сейчас в контексте агента |
| `/usage` | Расход токенов за сессию |
| `/resume` | Продолжить прошлый разговор после перезагрузки |
| `/help` | Справка и список всех доступных команд |

Выбор модели через `/` **не нужен** — Gateway AGIMA подбирает модель сам.

## Навигация по проекту

### Через символ `@`

Самый быстрый способ показать агенту конкретный файл или папку:

```
@company-context/SCENARIO.md расскажи суть
@inherited-chaos/ что здесь происходит
```

Символ `@` открывает fuzzy-поиск по файлам проекта. Не надо вводить полный путь — пиши кусок имени, VS Code подсказывает.

### Запросы без `@`

Для общих задач достаточно обычного текста:

```
Покажи файлы в текущей директории.
Что находится в src/components?
Есть ли в проекте тесты?
Покажи последние 5 коммитов.
```

## Работа с Git через агента

```
Покажи статус git.
Какие файлы изменены?
Сделай коммит с сообщением "add user authentication".
Создай новую ветку feature/user-profiles.
Покажи diff последнего коммита.
```

Агент сам подставит нужные `git`-команды и покажет вывод.

## Пользовательские команды

Пользовательские команды — это Markdown-файлы в папке `.claude/commands/`. Каждый такой файл становится `/имя-файла` в меню команд.

### Пример: команда `/test`

Создай файл `.claude/commands/test.md`:

```markdown
# Команда: Тесты

Запусти все тесты в проекте и проанализируй результаты.

## Шаги
1. Выполни `npm test`.
2. Проанализируй вывод.
3. Если есть ошибки — предложи исправления.
4. Сгенерируй краткий отчёт о покрытии.
```

Теперь в чате можно вызвать:

```
/test
```

Агент прочитает файл и выполнит шаги.

### Пример: команда `/review`

Файл `.claude/commands/review.md`:

```markdown
# Команда: Ревью кода

Проведи ревью изменений относительно main.

## Критерии
- Читаемость
- Обработка ошибок
- Безопасность
- Производительность
- Тестируемость

## Формат вывода
1. Общая оценка (1-10).
2. Найденные проблемы.
3. Рекомендации по улучшению.
```

После создания новых команд — перезагрузи окно VS Code (`Cmd/Ctrl+Shift+P → Developer: Reload Window`).

## Режимы работы агента

Агент работает в трёх режимах — переключай через `Shift+Tab`:

1. **Edit mode** (по умолчанию) — спрашивает разрешение перед каждым действием.
2. **Auto-accept mode** — выполняет без подтверждений. Быстрее, но требует доверия.
3. **Plan mode** — сначала показывает план, ты его смотришь, потом запускает.

**Bypass Permissions** — отдельный уровень: агент не спрашивает вообще ничего. Для рутинных задач опытным пользователям.

## Прерывание и откат

- **Escape** — прервать агента посреди действия.
- **Escape дважды** — откатить последние изменения (код и разговор, или только разговор).

## Think-ключевые слова

Если задача сложная, попроси агента думать дольше:

- `think about X` — нормальное мышление.
- `think harder` — глубокий анализ.
- `ultrathink` — максимальная мощность.

Расходует больше токенов — используй осмысленно.

## Полезные горячие клавиши VS Code

| Клавиша | Действие |
|---|---|
| `Cmd/Ctrl+P` | Quick Open — файл по имени |
| `Cmd/Ctrl+Shift+P` | Command Palette — любая команда VS Code |
| `Cmd/Ctrl+Shift+F` | Поиск по всему проекту |
| `Cmd/Ctrl+Shift+E` | Explorer (дерево файлов) |
| `` Cmd/Ctrl+` `` | Терминал |
| `Cmd/Ctrl+Shift+V` | Preview Markdown |

## Советы по эффективности

1. **`@файл`** — быстрее копипаста пути.
2. **`/compact`** — когда разговор стал длинным, а контекст забивается.
3. **Режим Plan** — для больших изменений: сначала смотришь шаги, потом запускаешь.
4. **Пользовательские команды** — для повторяющихся задач.
5. **Всё в git** — не бойся экспериментировать, откат в один клик.
