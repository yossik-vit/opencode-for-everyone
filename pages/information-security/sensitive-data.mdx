## 3.2: Чувствительные данные

- **Время прохождения:** 20 минут

> **Интерактивный урок:** Запустите `/start-3-2` в OpenCode

## Обзор

Конкретные списки: что нельзя передавать ИИ никогда, что можно после обезличивания, что безопасно использовать сразу.

**Ключевой вывод:** Три вопроса перед отправкой — содержат ли ПДн, может ли навредить, можно ли без этого.

## Красный список: никогда не отправляй

### Персональные данные (ПДн)
- ФИО клиентов, сотрудников, партнёров
- Номера телефонов, email-адреса
- Паспортные данные, СНИЛС, ИНН
- Адреса проживания, дата рождения

### Финансовые данные
- Номера банковских карт, реквизиты счетов
- Детализация платежей, зарплаты сотрудников

### Аутентификационные данные
- Пароли, API-ключи, токены
- SSH-ключи, криптографические ключи

### Коммерческая тайна
- Стратегические планы до объявления
- Списки клиентов и контракты

## Жёлтый список: обезличивай перед отправкой

| Данные | Что заменить | Пример замены |
|--------|-------------|---------------|
| Имена сотрудников | Имя → роль | «Иванов И.И.» → «Менеджер по продажам» |
| Названия клиентов | Название → код | «ООО Ромашка» → «Клиент А» |
| Адреса | Адрес → город | «ул. Ленина, 42»» → «Москва» |
| Суммы | Точная → диапазон | «1 234 567 руб.» → «около 1,2 млн» |

## Зелёный список: можно отправлять

- Общедоступная документация и стандарты
- Шаблоны документов без данных
- Описания процессов (без имён и сумм)
- Агрегированная статистика

## Частые ошибки

1. **Копировать-вставить из таблицы** — данные клиентов попадают в запрос
2. **Скриншот с чувствительными данными** — имена, номера, суммы на изображении
3. **«ИИ же забудет после сессии»** — нет гарантии, данные могут попасть в логи
