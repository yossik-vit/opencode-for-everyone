## Принцип минимального контекста

ИИ не нужна вся информация о проекте. Давай только то, что нужно для задачи.

### Плохо
> «Проанализируй продажи нашего филиала на Арбате. Менеджер — Сидоров. Средний чек 850₽, у Starbucks рядом — 920₽.»

Что попало: адрес, имя, внутренняя метрика, данные конкурента.

### Хорошо
> «Проанализируй данные по продажам кофейни (файл sales-feb.csv). Определи тренды и предложи 3 способа повысить средний чек.»

Тот же результат. Никаких чувствительных данных.

## Правила работы с файлами

### Перед отправкой файла в ИИ

1. **Проверь содержимое** — открой файл, просмотри данные
2. **Удали лишние столбцы** — только нужные для задачи
3. **Обезличь строки** — замени имена, адреса, номера
4. **Проверь скрытые данные** — в Excel бывают скрытые листы

### Скрытые риски по типам файлов

| Тип | Что проверить |
|-----|---------------|
| Excel | Скрытые листы, формулы, комментарии, метаданные автора |
| PDF | Метаданные, скрытые слои, цифровые подписи |
| Word | История изменений, комментарии, свойства |
| Изображения | EXIF-данные (геолокация, устройство) |
| CSV | Все столбцы — в данных может быть неожиданное |

## Безопасные шаблоны промптов

### Анализ данных
> «Проанализируй данные из файла [файл]. Определи тренды и аномалии. Данные обезличены.»

### Создание документа
> «Создай шаблон отчёта по продажам. Структура: заголовок, период, ключевые метрики, выводы. Без реальных данных.»

### Код
> «Напиши функцию для расчёта конверсии. Вход: количество посещений (int), количество покупок (int). Возвращает float.»

### Опасные паттерны — не делай так
- «Вот полная база клиентов, найди VIP»
- «Посмотри в config.env значения переменных»
- «Используй эти API-ключи для тестирования»

## Что делать при утечке

Если случайно отправил чувствительные данные:

1. Не паникуй — но действуй быстро
2. Сообщи ИБ-ответственному
3. Зафиксируй что отправлено
4. Не пытайся скрыть

Честный рапорт лучше скрытых последствий.
