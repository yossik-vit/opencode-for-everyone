# Какие данные нельзя передавать ИИ

## Красный список: никогда не отправляй

### Персональные данные (ПДн)
- ФИО клиентов, сотрудников, партнёров
- Номера телефонов, email-адреса
- Паспортные данные, СНИЛС, ИНН
- Адреса проживания
- Дата рождения, семейное положение
- Биометрические данные

### Финансовые данные
- Номера банковских карт
- Реквизиты счетов
- Детализация платежей
- Бухгалтерская отчётность до публикации
- Зарплаты сотрудников

### Аутентификационные данные
- Пароли и хеши паролей
- API-ключи и токены
- Секреты OAuth
- SSH-ключи
- Криптографические ключи

### Коммерческая тайна
- Стратегические планы до объявления
- Данные о слияниях и поглощениях
- Непатентованные изобретения
- Списки клиентов и контракты
- Ценовая политика до утверждения

### Юридические данные
- Договоры до подписания
- Судебные документы
- Данные, подпадающие под NDA
- Регуляторные отчёты до сдачи

## Жёлтый список: обезличивай перед отправкой

Эти данные можно использовать, но только после удаления или замены чувствительных элементов:

| Данные | Что заменить | Пример замены |
|--------|-------------|---------------|
| Имена сотрудников | Имя → роль | «Иванов И.И.» → «Менеджер по продажам» |
| Названия клиентов | Название → код | «ООО Ромашка» → «Клиент А» |
| Адреса | Адрес → город | «ул. Ленина, 42» → «Москва» |
| Даты | Точная дата → период | «15.03.2024» → «март 2024» |
| Суммы | Точная сумма → диапазон | «1 234 567 руб.» → «около 1,2 млн» |

## Зелёный список: можно отправлять

- Общедоступная документация
- Технические стандарты и спецификации
- Шаблоны документов без данных
- Описания процессов (без имён и сумм)
- Примеры кода без ключей и паролей
- Агрегированная статистика (без персональных данных)

## Практическое правило

Задайте себе три вопроса перед отправкой:

1. **Содержит ли запрос персональные данные?** Если да — обезличить или не отправлять.
2. **Может ли эта информация навредить компании при утечке?** Если да — не отправлять.
3. **Могу ли я решить задачу без этих данных?** Если да — убери лишнее.

## Частые ошибки

### Ошибка 1: Копировать-вставить из таблицы
Вы копируете таблицу с данными клиентов и вставляете в ИИ для анализа. Даже если вы просите «проанализировать тренды», все данные попадают в запрос.

**Решение:** Экспортируйте агрегированные данные или замените персональные поля.

### Ошибка 2: Скриншот с чувствительными данными
Вы отправляете скриншот экрана, на котором видны имена, номера, суммы.

**Решение:** Затрите чувствительные данные на скриншоте перед отправкой.

### Ошибка 3: «ИИ же забудет после сессии»
Нет гарантии, что данные не будут использованы для обучения модели или не попадут в логи.

**Решение:** Работайте так, будто каждый запрос может стать публичным.
