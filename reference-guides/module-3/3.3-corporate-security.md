# Безопасная работа с корпоративной информацией

## Почему это отдельная тема

Корпоративные данные — это не только пароли. Это внутренние процессы, переписка, структура команд, бизнес-метрики. Даже безобидные на вид данные в совокупности могут дать конкурентам или злоумышленникам полную картину о вашей организации.

## Принцип минимального контекста

ИИ не нужна вся информация о вашем проекте. Давайте только то, что необходимо для решения задачи.

### Плохо
> «Проанализируй продажи нашего филиала на Арбате за февраль. У нас там работает 12 человек, менеджер — Сидоров Алексей. Средний чек 850 рублей, у Starbucks на соседней улице — 920.»

Что попало в запрос: адрес филиала, имя сотрудника, внутренняя метрика, данные о конкуренте.

### Хорошо
> «Проанализируй данные по продажам кофейни (файл sales-feb.csv). Определи тренды и предложи 3 способа повысить средний чек.»

Тот же результат, но без чувствительных данных.

## Правила работы с файлами

### Перед отправкой файла в ИИ

1. **Проверь содержимое** — открой файл и просмотри данные
2. **Удали лишние столбцы** — оставь только нужные для задачи
3. **Обезличь строки** — замени имена, адреса, номера
4. **Проверь скрытые данные** — в Excel могут быть скрытые листы, в PDF — метаданные

### Что проверять в разных типах файлов

| Тип файла | Скрытые риски |
|-----------|---------------|
| **Excel** | Скрытые листы, формулы со ссылками на другие файлы, комментарии, метаданные автора |
| **PDF** | Метаданные, скрытые слои, комментарии, цифровые подписи |
| **Word** | История изменений, комментарии, свойства документа |
| **Изображения** | EXIF-данные (геолокация, устройство), водяные знаки |
| **CSV** | Проверь все столбцы — иногда в данных есть то, что не ожидаешь |

## Работа с промптами

### Безопасные шаблоны промптов

**Анализ данных:**
> «Проанализируй данные из файла [файл]. Определи основные тренды и аномалии. Данные обезличены.»

**Создание документа:**
> «Создай шаблон отчёта по продажам. Используй структуру: заголовок, период, ключевые метрики, выводы. Без реальных данных.»

**Код:**
> «Напиши функцию для расчёта конверсии. Входные параметры: количество посещений (int), количество покупок (int). Возвращает float.»

### Опасные паттерны промптов

Не делайте так:

- «Вот полная база клиентов, найди VIP-клиентов»
- «Посмотри в файле config.env значения переменных»
- «Используй эти API-ключи для тестирования»
- «Проанализируй переписку сотрудников из этого чата»

## Если произошла утечка

Если вы случайно отправили чувствительные данные:

1. **Не паникуй** — но действуй быстро
2. **Сообщи ИБ-ответственному** — даже если кажется мелочью
3. **Зафиксируй что отправлено** — скриншот или описание
4. **Не пытайся скрыть** — лучше честный рапорт, чем последствия скрытой утечки

## Чек-лист перед работой с ИИ

- [ ] Данные обезличены (имена, номера, адреса заменены)
- [ ] Файл проверен на скрытые данные
- [ ] В запросе только информация, необходимая для задачи
- [ ] Нет паролей, ключей, токенов
- [ ] Нет юридически чувствительной информации
- [ ] Если сомневаюсь — спросил у руководителя или ИБ
