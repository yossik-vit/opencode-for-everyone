# Итеративная разработка

## Переключение на режим сборки

```
> /build
# или нажмите Tab
```

## Итерация 1: Базовый CRUD

### Шаг 1: Создание типов

```
> Создай файл src/types.ts с типами для todo приложения:
> - Todo интерфейс
> - Filter тип
> - TodoState
> - TodoAction для reducer
```

### Шаг 2: Создание хука useTodos

```
> Создай хук src/hooks/useTodos.ts:
> - useReducer для управления состоянием
> - Функции: addTodo, deleteTodo, toggleTodo, updateTodo
> - Экспорт состояния и действий
```

### Шаг 3: Создание TodoForm

```
> Создай компонент src/components/TodoForm.tsx:
> - Input для ввода задачи
> - Валидация (не пустое)
> - Submit по Enter
> - Очистка после отправки
> - CSS Modules для стилей
```

### Шаг 4: Создание TodoItem

```
> Создай компонент src/components/TodoItem.tsx:
> - Чекбокс для отметки выполнения
> - Текст задачи
> - Кнопка удаления
> - Inline редактирование по двойному клику
> - CSS Modules для стилей
```

### Шаг 5: Создание TodoList

```
> Создай компонент src/components/TodoList.tsx:
> - Список TodoItem
> - Empty state если нет задач
> - CSS Modules для стилей
```

### Шаг 6: Интеграция в App

```
> Обнови src/App.tsx:
> - Используй useTodos хук
> - Добавь TodoForm и TodoList
> - Добавь базовые стили
```

### Шаг 7: Тесты

```
> Напиши тесты для useTodos хука
```

## Итеративный подход с AI-агентом

### Правильный порядок

1. **Планирование** — спланируйте функционал
2. **Сборка** — реализуйте
3. **Тестирование** — проверьте
4. **Коммит** — сохраните
5. **Повтор** — следующая итерация

### Пример итерации

```
# 1. Планирование
> /plan
> Спроектируй компонент для drag & drop сортировки задач

# 2. Сборка
> /build
> Реализуй drag & drop для TodoList

# 3. Тестирование
> Запусти тесты и проверь что всё работает

# 4. Коммит
> Сделай коммит с сообщением "add drag and drop sorting"
```
