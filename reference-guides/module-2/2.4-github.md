# Интеграция с GitHub

## Настройка Git

### Проверка конфигурации

```bash
git config user.name
git config user.email
```

### Настройка (если нужно)

```bash
git config --global user.name "Ваше имя"
git config --global user.email "your@email.com"
```

## Создание репозитория на GitHub

### Через GitHub CLI (рекомендуется)

```bash
# Установка GitHub CLI
brew install gh

# Авторизация
gh auth login

# Создание репозитория
gh repo create todo-app --public --source=. --remote=origin --push
```

### Через AI-агента

```
> Создай репозиторий на GitHub для этого проекта
> Используй gh cli
```

### Через веб-интерфейс

1. Откройте github.com/new
2. Введите имя репозитория
3. Выберите Public/Private
4. Нажмите Create repository

## Git Flow

### Структура веток

```
main (продакшн)
├── develop (разработка)
├── feature/* (функционал)
├── bugfix/* (исправления)
└── release/* (релизы)
```

### Работа с Git Flow

```
> Создай ветку feature/authentication из develop
> ... работа ...
> Слишь feature/authentication в develop
> Создай PR из develop в main
```

## Автоматизация с GitHub Actions

### Создание workflow

```
> Создай GitHub Actions workflow для:
> - Запуска тестов при PR
> - Линтинга кода
> - Деплоя при merge в main
```

### Пример workflow

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run lint
      - run: npm test
      - run: npm run build
```
