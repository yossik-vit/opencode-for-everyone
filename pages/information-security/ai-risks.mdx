## 3.5: Риски использования ИИ в рабочих задачах

- **Время прохождения:** 20 минут

> **Интерактивный урок:** Запустите `/start-3-5` в OpenCode

## Обзор

Финальный урок модуля. Шесть главных рисков использования ИИ в работе и конкретные способы защиты.

**Ключевой вывод:** Проверяй, обезличивай, минимизируй, критикуй, сообщай, учись.

## Шесть рисков

### 1. Утечка данных через контекст
Данные загружаются в облачный ИИ и побывают на серверах третьей стороны.

**Защита:** Локальные модели, opt-out обучения, обезличивание.

### 2. Галлюцинации
ИИ генерирует правдоподобный, но неверный ответ.

**Защита:** Проверяй факты в первоисточниках. Не используй ИИ как единственный источник.

### 3. Переоценка возможностей
Доверяете ИИ задачи, требующие человеческого суждения.

**Защита:** ИИ — помощник для вариантов. Человек — для решений.

### 4. Зависимость от ИИ
Собственные навыки атрофируются.

**Защита:** Понимай, что делает ИИ. Периодически работай без него.

### 5. Нарушение лицензий
ИИ генерирует код на основе чужих работ без проверки лицензии.

**Защита:** Проверяй лицензии сгенерированного кода.

### 6. Prompt injection
Скрытые指令 в документах из внешних источников.

**Защита:** Проверяй документы, не загружай из сомнительных источников.

## Матрица рисков

| Риск | Вероятность | Влияние | Приоритет |
|------|-------------|---------|-----------|
| Утечка данных | Средняя | Высокий | Критичный |
| Галлюцинации | Высокая | Средний | Высокий |
| Переоценка | Высокая | Средний | Высокий |
| Зависимость | Средняя | Низкий | Средний |
| Нарушение лицензий | Низкая | Высокий | Средний |
| Prompt injection | Низкая | Высокий | Средний |

## Золотые правила

1. **Проверяй** — не доверяй ИИ на слово
2. **Обезличивай** — убирай персональные данные
3. **Минимизируй** — только необходимая информация
4. **Критикуй** — оценивай ответы с позиции здравого смысла
5. **Сообщай** — при подозрении на утечку или атаку
6. **Учись** — угрозы меняются, обновляй знания
