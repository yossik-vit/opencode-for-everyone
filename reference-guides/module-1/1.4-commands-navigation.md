# Команды и навигация

## Основные команды

### Команды чата

| Команда | Описание |
|---------|----------|
| `/help` | Показать справку |
| `/clear` | Очистить историю чата |
| `/exit` или `/quit` | Выйти из Claude Code |
| `/model` | Показать/изменить текущую модель |
| `/config` | Показать текущую конфигурацию |

### Управление моделями

```
# Показать текущую модель
> /model

# Переключить модель
> /model gpt-4o
> /model deepseek/deepseek-r1
> /model anthropic/claude-sonnet-4-20250514

# Список доступных моделей
> /models
```

### Управление сессиями

```
# Новая сессия
> /new

# История сессий
> /history

# Загрузить сессию
> /load session-123
```

```

> /config show

> /config edit

```

## Пользовательские команды

### Создание команд

Создайте `.md` файлы в директории `.claude/commands/`:

```bash
mkdir -p .claude/commands
```

### Пример: Команда для тестов

```markdown
<!-- .claude/commands/test.md -->
# Команда: Тесты

Запусти все тесты в проекте и проанализируй результаты.

## Шаги:
1. Выполни `npm test`
2. Проанализируй вывод
3. Если есть ошибки, предложи исправления
4. Сгенерируй отчёт о покрытии
```

### Использование пользовательской команды

```
> /test
```

### Пример: Команда для ревью кода

```markdown
<!-- .claude/commands/review.md -->
# Команда: Ревью кода

Проведи ревью кода в текущей директории.

## Критерии:
- Читаемость кода
- Обработка ошибок
- Безопасность
- Производительность
- Тестируемость

## Формат вывода:
1. Общая оценка (1-10)
2. Найденные проблемы
3. Рекомендации по улучшению
```

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `Tab` | Переключить режим (сборка/планирование) |
| `Enter` | Отправить сообщение |
| `Shift+Enter` | Новая строка |
| `Ctrl+C` | Отменить/Выйти |
| `Ctrl+L` | Очистить экран |
| `↑` / `↓` | Навигация по истории |
| `Ctrl+R` | Поиск по истории |

## Работа с длинными выводами

### Прокрутка

```
> Покажи весь файл src/index.ts
# Используйте ↑/↓ для прокрутки
```

### Копирование

```
# Скопировать вывод в буфер обмена
> Покажи содержимое package.json и скопируй в буфер
```

### Экспорт

```
# Экспорт в файл
> Сохрани результат анализа в файл analysis.md
```

## Режимы ввода

### Однострочный ввод

```
> Объясни эту функцию
```

### Многострочный ввод

```
> Напиши тест для этой функции:
> function add(a: number, b: number): number {
>   return a + b;
> }
```

### Вставка кода

```
> Проанализируй этот код:
> ```
> const data = await fetch(url);
> const json = await data.json();
> return json;
> ```
```

```

### Терминальные команды

```
> Выполни ls -la в текущей директории

```

## Советы по эффективности

1. **Используйте Tab** — для быстрого переключения режимов
2. **Команды через /** — для системных операций
3. **История через ↑** — для повторения запросов
4. **Пользовательские команды** — для часто повторяющихся задач
5. **Горячие клавиши** — для ускорения работы
