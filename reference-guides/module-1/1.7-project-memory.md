# Память проекта (AGENTS.md)

## Что такое AGENTS.md?

Файл `AGENTS.md` — инструкция для Claude Code о вашем проекте. Работает как долгосрочная память ассистента.

## Почему это важно?

Без AGENTS.md:
- Claude Code каждый раз угадывает стек технологий
- Не знает соглашений проекта
- Может предложить несовместимые решения

С AGENTS.md:
- Claude Code понимает контекст проекта
- Следует вашим соглашениям
- Использует правильные инструменты
- Помогает новым участникам команды

## Создание AGENTS.md

### Базовый шаблон

```markdown
# Инструкции для Claude Code

## О проекте
Краткое описание проекта и его целей.

## Стек технологий
- Язык: TypeScript
- Фреймворк: React
- База данных: PostgreSQL
- ORM: Prisma

## Соглашения
- Используй TypeScript строгую типизацию
- Пиши тесты для всех функций
- Следуй ESLint конфигурации
- Используй Prettier для форматирования

## Структура проекта
/src
  /components   # React компоненты
  /utils        # Утилиты и хелперы
  /api          # API клиенты
  /types        # TypeScript типы
/tests          # Тесты
/docs           # Документация

## Команды
- `npm run dev` — запуск dev сервера
- `npm test` — запуск тестов
- `npm run lint` — проверка стиля
- `npm run build` — сборка проекта

## Важные замечания
- Не используй `any` тип
- Все функции должны иметь JSDoc
- Коммиты следуют Conventional Commits
```

## Расположение файла

### Варианты расположения

1. **Корень проекта** (рекомендуется)
```
my-project/
├── AGENTS.md
├── src/
├── package.json
└── ...
```

2. **В скрытой директории**
```
my-project/
├── .claude/
│   └── AGENTS.md
├── src/
└── ...
```

3. **Глобальная конфигурация**
```
~/.claude/
└── AGENTS.md  # Общие инструкции для всех проектов
```

## Примеры AGENTS.md

### Для React проекта

```markdown
# Инструкции для Claude Code

## О проекте
E-commerce платформа на React с TypeScript.

## Стек
- React 18 с хуками
- TypeScript (строгий режим)
- Redux Toolkit для состояния
- React Router для навигации
- Styled Components для стилей
- Jest + React Testing Library для тестов

## Соглашения
- Компоненты: PascalCase (UserCard.tsx)
- Утилиты: camelCase (formatDate.ts)
- Тесты: *.test.tsx рядом с компонентом
- Стили: *.styles.ts рядом с компонентом

## Структура
/src
  /components    # Переиспользуемые компоненты
  /features      # Фичи (components + hooks + slice)
  /pages         # Страницы
  /hooks         # Кастомные хуки
  /services      # API сервисы
  /store         # Redux store
  /types         # TypeScript типы
  /utils         # Утилиты

## Правила
1. Все компоненты должны быть функциональными
2. Используй хуки вместо HOC
3. Каждый feature в своей директории
4. Экспортируй через barrel файлы (index.ts)
5. Тестируй через user-centric queries

## API соглашения
- Все запросы через services/api.ts
- Используй RTK Query для кэширования
- Обрабатывай ошибки в middleware
```

### Для Node.js бэкенда

```markdown
# Инструкции для Claude Code

## О проекте
REST API для управления пользователями и заказами.

## Стек
- Node.js 20
- Express.js
- TypeScript
- PostgreSQL + Prisma
- Jest для тестов
- Docker для развертывания

## Архитектура
- Clean Architecture
- Repository pattern
- Dependency injection

## Структура
/src
  /controllers   # Express контроллеры
  /services      # Бизнес логика
  /repositories  # Работа с БД
  /models        # Модели данных
  /middleware     # Express middleware
  /utils         # Утилиты
  /config        # Конфигурация
  /types         # TypeScript типы

## Конвенции
- Контроллеры: *Controller.ts
- Сервисы: *Service.ts
- Репозитории: *Repository.ts
- Все async/await (без .then)
- Валидация через Zod
- Логирование через Winston

## Безопасность
- JWT аутентификация
- Rate limiting
- Input validation
- SQL injection prevention (Prisma)
- CORS настройка

## API стандарты
- RESTful endpoints
- JSON ответы
- HTTP status codes
- Pagination через query params
- Error response format:
  ```json
  {
    "error": {
      "code": "NOT_FOUND",
      "message": "User not found"
    }
  }
  ```
```

### Для Python проекта

```markdown
# Инструкции для Claude Code

## О проекте
Data pipeline для обработки и анализа данных.

## Стек
- Python 3.11
- FastAPI для API
- SQLAlchemy для ORM
- Celery для фоновых задач
- PostgreSQL
- Redis для кэша

## Структура
/app
  /api           # FastAPI роуты
  /core          # Конфигурация и настройки
  /models        # SQLAlchemy модели
  /schemas       # Pydantic схемы
  /services      # Бизнес логика
  /tasks         # Celery задачи
  /utils         # Утилиты
/tests           # Тесты

## Соглашения
- Type hints везде
- Docstrings (Google style)
- Black для форматирования
- Ruff для линтинга
- Pytest для тестов

## Качество кода
- Максимум 80 символов в строке
- Импорты отсортированы (isort)
- Нет unused imports
- Все функции типизированы
```

## Обновление AGENTS.md

### Когда обновлять
- Новые технологии в проекте
- Изменение соглашений
- Новые команды
- Архитектурные изменения

### Как обновлять

```
> Обнови AGENTS.md: добавь информацию о новом сервисе уведомлений
> в секцию /services и обнови зависимости
```

## Множественные AGENTS.md

### Иерархия файлов

```
~/.claude/AGENTS.md              # Глобальные настройки
my-project/AGENTS.md               # Настройки проекта
my-project/.claude/AGENTS.md     # Дополнительные настройки
```

Приоритет: более специфичные файлы перезаписывают общие.

## Советы

1. **Начните с простого** — базовый шаблон лучше, чем ничего
2. **Обновляйте регулярно** — документируйте изменения
3. **Делитесь с командой** — commit AGENTS.md в репозиторий
4. **Используйте примеры** — добавьте конкретные примеры кода
5. **Будьте конкретны** — "TypeScript" лучше, чем "современный стек"
